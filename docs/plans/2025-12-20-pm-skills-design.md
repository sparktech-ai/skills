# PM Skills 设计文档

## 概述

为独立开发者设计的产品经理工作流 skill，覆盖从市场调研到 PRD 撰写的完整流程。

## Skill 列表

| Skill | 触发场景 | 输入 | 产出 |
|-------|---------|------|------|
| `/research` | 有想法或想找方向时 | 目的、想法（可空） | `docs/prd/research-<主题>.md` |
| `/prd` | 想法明确，准备写 PRD | 想法或调研文档 | `docs/prd/<产品名>.md` |

## 文件结构

```
skills/
├── research/
│   └── SKILL.md
└── prd/
    └── SKILL.md

docs/prd/
├── research-<主题>.md   # /research 产出
└── <产品名>.md          # /prd 产出
```

## `/research` Skill

### 目标

帮助独立开发者从模糊想法（甚至空白）到明确的产品方向。

### 流程

**阶段 1：理解目标**
1. 询问目的（变现、练手、解决自己痛点等）
2. 询问是否有初步想法或完全空白
3. 约束条件（可选，不主动追问）

**阶段 2：市场探索**
- 使用 WebSearch 调研热门赛道、趋势
- 根据目的筛选适合独立开发者的机会
- 产出 2-3 个潜在方向
- **主动推荐**：给出排序和理由，不只是罗列

**阶段 3：深度调研（针对选定方向）**
- 竞品分析：主要玩家、优劣势
- 用户画像：目标用户、核心痛点
- 市场机会：切入点、差异化空间

**阶段 4：产出文档**

生成 `docs/prd/research-<主题>.md`，结构：

```markdown
# <主题> 市场调研

## 目标与背景
- 目的
- 约束（如有）

## 市场机会
- 赛道分析
- 趋势

## 竞品分析
- 主要玩家
- 优劣势对比

## 用户画像与痛点
- 目标用户
- 核心痛点

## 建议方向
- 推荐切入点
- 理由
```

### 关键原则

- 当用户想法模糊时，Claude 主动调研并给出推荐方案
- 推荐需有理由，用户决定接受、调整或拒绝

## `/prd` Skill

### 目标

将产品想法转化为结构化的 PRD 文档。

### 流程

**阶段 1：确定输入**
- 检查 `docs/prd/research-*.md` 是否存在
- 有则询问是否基于该调研继续
- 无则请用户描述产品想法

**阶段 2：产品定义（对话引导）**
- 核心价值：解决什么问题？
- 目标用户：谁会用？
- 核心场景：用户怎么用？
- MVP 范围：第一版必须有什么？

**阶段 3：细化需求**
- 功能需求，按阶段划分：
  - 阶段 1（验证期）：最小功能 + 验证方式 + 成功标准
  - 阶段 2（完善期）：验证通过后的补充功能
  - 阶段 3（扩展期，可选）
- 非功能需求（性能、安全等必要项）
- 约束条件（必要的技术/资源限制）

**阶段 4：产出文档**

生成 `docs/prd/<产品名>.md`，结构：

```markdown
# <产品名> PRD

## 产品概述
- 一句话描述
- 目标用户
- 核心价值

## 用户故事 / 使用场景

## 功能需求

### 阶段 1：验证期
- 功能列表
- 验证方式
- 成功标准

### 阶段 2：完善期
- 功能列表

### 阶段 3：扩展期（可选）
- 功能列表

## 非功能需求

## 约束条件

## 成功指标
```

### 关键原则

- 灵活输入：有调研文档则读取，无则从想法开始
- 当用户不确定时，Claude 基于调研结果主动推荐方案
- 聚焦产品层面，技术只写必要约束
